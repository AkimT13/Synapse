01 / The pipeline

Two inputs. One searchable substrate.

Code and documentation live in different grammars. Synapse collapses them into the same natural-language surface before embedding, so a query doesn't have to know which one it's looking for.

Dual ingestion

Python source is parsed via AST — functions, signatures, decorators, docstrings preserved. PDFs, DOCX, Markdown and HTML flow through docling's hybrid chunker. One schema, two pipelines.

ingestion / code + knowledge

LLM-normalized chunks

Every function and every paragraph is rewritten into a natural-language description. Embeddings of code and prose land in the same region of the vector space — no retrieval gap between modalities.

normalization · openai or ollama

Cross-modal retrieval

A single Actian VectorAI collection, cosine-ranked. Ask a code question and get the spec; ask a domain question and get the implementation. Payloads carry full lineage back to the source file.

storage · actian vectorai

02 / Integration

Four lines from repo to retrieval.

The ingestion job is a single object. Connect a VectorStore, point it at a directory, and every function in the tree becomes a queryable vector with lineage intact.

  • —Deterministic SHA-256 IDs, so re-ingesting doesn't duplicate.
  • —Batched upserts via Actian's SmartBatcher.
  • —Per-stage error capture — partial failure returns a usable JobResult.
jobs/ingest_code.py 
from storage.vector_store import VectorStore
from jobs.ingest_code   import CodeIngestionJob

# Walk → normalize → embed → upsert, in one pass.
with VectorStore() as store:
    result = CodeIngestionJob(store).run("/path/to/repo")
    print(f"stored {result.chunks_stored} chunks "
          f"from {result.files_processed} files")
